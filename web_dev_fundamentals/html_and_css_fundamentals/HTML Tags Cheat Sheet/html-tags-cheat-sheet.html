Headings

Headings from `h1` to `h6` identify blocks and sections. Every page should have one `h1`. While technically not required, it is important for SEO.

Paragraphs

Paragraphs of text go into `p` tags.

Lists

Unordered Lists

Unordered lists are marked with bullet points and go in `ul` tags. Each list item goes in an `li` tag.

Types of lists:

  • Unordered lists (`ul`)
  • Ordered lists (`ol`)

Ordered Lists

Ordered lists are numbered and go in `ol` tags. Each list item goes in `ol` tags. Each list item goes in an `li` tag.

Anchors (i.e. links)

Anchor tags create links to other pages. The URL goes in the `href` attribute. Go to DuckDuckGo to search for more information.

Setting `target="_blank"` will open the link in a new window. See MDN for more details.

Anchor tags can also create links to a new location on the same page. Instead of putting a URL in the `href` attribute, just use the `id` of the element you want to navigate to with a `#` at the start. This allows you to jump to the top of the page.

Text Emphasis

Italic

The `em` element adds extra emphasis to the text within the opening and closing tags. This shows up as italic text in most browsers.

Bold

You can also use the `strong` element to emphasize text. This shows up as bold text in most browsers.

Code

You can also use the `code` element to differentiate that certain text is to be interpreted as a block of code. These blocks can be styled by css but in most browsers by default will show in a specific monospaced font. It is common to style these blocks such that the background ia different colour to regular text.

Breaking text using the `br` tag

You can also use the `br` tag to create a line break deliberately. Unlike many other tags it is a single tag without a matching closing tag. It is used equivalently to how you would use a newline character (`\n`) in other code.
